## Правило: проверяй 7dots перед реализацией

**ОБЯЗАТЕЛЬНО** перед реализацией любой функциональности в `.rpy` файлах — вызвать skill `7dots-reference` и убедиться, что нужного нет в библиотеке. Это касается: показ/скрытие спрайтов, аудио (музыка/звуки/голос), переходы, движение персонажей, эффекты размытия/цвета/яркости, UI-элементы, анимации, управление диалогом. Если функция есть в 7dots — используй её, не дублируй.

## Project Overview

**Шаркающий человек (Shuffling Man)** — психологический хоррор на движке Ren'Py 8.5.3. Нарратив от ненадёжного рассказчика (Марина): история о распаде семьи и психическом расстройстве. Визуальная реальность постепенно разрушается — от "идеализированной" версии событий до мрачной правды. Сеттинг: Москва, начало 2000-х.

## Architecture

### File Organization

```
game/
├── script.rpy       — точка входа, весь нарратив (сейчас stub)
├── options.rpy      — конфиг движка (имя, звук, переходы)
├── gui.rpy          — визуальные параметры UI (1920×1080, акцент #cc0000)
├── screens.rpy      — все экраны интерфейса (диалог, меню, сохранения)
└── libs/
    ├── libs.txt     — заставляет Ren'Py загружать libs/ перед остальными файлами
    └── 7dots.rpy    — расширенная библиотека утилит (загружается до всего)
```

Черновые ассеты (PSD/JPG) живут вне репозитория: `../shuffling_man_assets/Черновик/`.

### Load Order

Ren'Py загружает файлы в следующем порядке благодаря `libs/libs.txt`:
1. `libs/7dots.rpy` (init приоритеты -999 → 1900)
2. `gui.rpy` (offset -2)
3. `screens.rpy` (offset -1)
4. `script.rpy` и остальные `.rpy`-файлы

### 7dots Library

`game/libs/7dots.rpy` — основной инструментарий проекта. Покрывает: аудио (mplay/splay/vplay/sfxplay), трансформации (позиции, масштаб, цвет, силуэты, 3D-повороты, анимации), систему времён суток, управление диалогом (dismiss_on/off), спрайт-утилиты, переходы (Flash, TurnPage).

**Перед реализацией любой функциональности используй skill `7dots-reference`** — полный справочник всех функций с примерами.

## Конвенции

Полный набор конвенций — в README.md, раздел «Конвенции». Он является источником истины; ниже — ключевые правила.

### Файлы и папки

- Имена файлов и папок — строго на английском, нижний регистр, слова через `_`.
- Папки внутри `game/` — максимум один уровень вложенности (папка на главу, `endings/` для концовок).
- Имена файлов избыточны и полностью дублируют структуру повествования (файл идентифицируется по имени без учёта пути):
  - Сцены: `prologue/prologue_scene_1.rpy`, `chapter_1/chapter_1_scene_2.rpy`
  - Концовки: `endings/ending_1.rpy`
  - Мини-игры: в папке своей главы — `chapter_2/chapter_2_scene_3_minigame_piano_1.rpy`
- Код делить на файлы аккуратно, без избыточного дробления.
- При удалении/переименовании `.rpy` удалять осиротевший `.rpyc`.

### Код

- `define` — константы (персонажи, transforms); `default` — все изменяемые игровые переменные (флаги, счётчики). Игровые флаги никогда не инициализировать через `$` или `init python`.
- Предпочитать операторы Ren'Py (`show`, `play`, `while`, `menu`) их Python-эквивалентам — их проверяет lint и предсказывает движок.
- Глобальные лейблы уникальны и дублируют структуру по той же схеме, что имена файлов: `chapter_1_scene_2`, `prologue_scene_1`, `ending_1`; точки внутри сцены — локальные лейблы (`.loop`).
- У `call` в релизе должен быть `from`-клаус.
- Все строки интерфейса — в `_("...")`. Игра пишется на русском (язык None).

### Мультиязычность (подробно — README, «Мультиязычность»)

- Каждая видимая игроку литеральная строка (экраны, кнопки, `Character(_("..."))`, `renpy.notify`, мини-игры) — в `_("...")`.
- Строки не склеивать — подстановки `[var]`; переведённое значение — `[var!t]`; одинаковые строки с разным смыслом — `{#контекст}`.
- Логика сравнивает ключи, не отображаемый текст; в сейвы/persistent — ключи, не строки.
- Мини-игры: текст не «запекать» в картинки (иначе — вариант файла в `game/tl/<язык>/...`); верстать с запасом по ширине; служебные лейблы — с `hide`.
- Шрифты только через `gui.text_font`/стили; на язык — `translate <язык> python`/`style`-блоки без побочных эффектов.

### Ассеты

- Изображения — `game/images/`, имена в нижнем регистре латиницей, формат «тег атрибут» через пробел (исключение из правила `_`): `marina sad.png`, `bg kitchen day.jpg`.
- Подпапки в `game/images/` обязательны: по главам (`prologue/`, `chapter_1/`, …), при необходимости — по сценам; сквозные ассеты (персонажи, UI) — в `common/`. Имя образа берётся из имени файла, путь движок игнорирует → имена уникальны во всём `game/images/`.
- Аудио — `game/audio/`, имена в нижнем регистре, слова через `_`, без пробелов и дефисов.

### Аудио: планируемая своя система

Штатных каналов Ren'Py недостаточно для задуманного саунд-дизайна. Планируется собственная система: по 5 буферных каналов на каждую линию (`music`/`voice`/`sfx`) для наложений и кроссфейдов. Её дизайн будет обсуждаться отдельно — до утверждения ничего не изобретать самостоятельно. Пока временно: аудио только через обёртки 7dots (`mplay`/`splay`/`sfxplay`/`vplay`), никаких голых `play`/`stop` в сценариях — так миграция сведётся к замене реализации обёрток.
