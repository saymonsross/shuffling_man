# Правило: состояние function-трансформов — только в `_fx_state`

Касается любых function-трансформов с накапливаемым состоянием (сглаживание к цели, дрожь, плавная альфа и т.п.).

- Состояние хранить ТОЛЬКО в module-level словаре `_fx_state` (`game/common/camera_fx.rpy`) через `_fx_step`, под уникальным ключом (параметр `key` у трансформа/функции).
- НЕ хранить состояние в атрибутах `trans` (`trans.fx_*`): camera-слой и always_shown-экраны пересобирают обёртку трансформа при каждом `restart_interaction` (любой клик/наведение) — атрибуты теряются, эффект даёт видимый рывок.
- НЕ сбрасывать состояние по `st ≈ 0`: повторное применение трансформа (`camera at ...` на стыке сцен, повторный `show ... at ...`) запускает ATL заново, и сброс даёт скачок эффекта к нулю и обратно — «вздрагивание» камеры на переходах. Накопитель должен продолжаться с сохранённого значения; `start` в `_fx_step` — только для самого первого обращения к ключу.
- Ключи уникальны на каждого одновременного пользователя эффекта: камера — `"cam"`, объектам — свои ключи (`"gg"`, `jitter_key` и т.п.). Общий ключ у двух одновременно живущих трансформов = перетягивание одного накопителя.
- Проверка при ревью нового эффекта: grep на `trans.fx` и `st <` в function-трансформах — совпадений быть не должно.
