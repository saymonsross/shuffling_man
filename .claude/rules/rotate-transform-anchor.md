# Правило: `rotate`/`zoom` с якорным позиционированием — только с `transform_anchor True`

- Любой трансформ, где есть `rotate` (в том числе анимируемый в `parallel`-треке) И позиционирование спрайта через `anchor`/`pos`, обязан содержать `transform_anchor True`.
- Причина: активный `rotate` с дефолтным `rotate_pad` расширяет холст спрайта до квадрата со стороной, равной диагонали (у 363×793 → ≈872×872). Без `transform_anchor` свойства `anchor`/`pos` отсчитываются от этого квадрата, а не от спрайта — объект уезжает с позиции, «размер» в инспекторе выглядит как N×N с одинаковыми сторонами.
- С `transform_anchor True` якорь считается по самому спрайту и одновременно служит пивотом поворота и масштаба — выбирать точку якоря осмысленно: крепление маятника — `anchor (0.5, 1.0)` в точке оси, покачивание корпуса персонажа — низ фигуры по центру.
- Эталонные примеры в проекте: `c1s1_arrow_swing` (стрелка метронома) и `c1s1_gg_idle` (ГГ за пианино) в `game/1_chapter/chapter_1_scene_1.rpy`.
- Проверка при ревью: grep `rotate` по `.rpy` — в каждом найденном трансформе с `anchor`/`pos` должен стоять `transform_anchor True` (кроме случаев, когда позиционирование идёт через `align`/`xalign`+`yalign` по центру и сдвиг холста не важен).
